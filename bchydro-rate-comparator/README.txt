BC Hydro Residential Rate Comparator

What it does

This local website compares annual electricity costs for these residential options:

- RS 1101, Residential Service, Tiered Rate
- RS 1101 + RS 2101, Tiered Rate with Time-of-Day
- RS 1151, Residential Service, Flat Rate
- RS 1151 + RS 2101, Flat Rate with Time-of-Day

The app accepts BC Hydro consumption CSV files formatted with the following columns:

Account Holder, Account Number, Meter Number, Interval Start Date/Time, Time of Day Period, Inflow (kWh), Outflow (kWh), Net Consumption (kWh), Demand (kW), Power Factor (%), Estimated Usage, Service Address, City

Build

The browser app is authored in TypeScript:

- src/app.ts is the source file.
- app.js is generated by npm run build and is the file loaded by index.html.
- dist/ is generated by npm run build:pages and is the static GitHub Pages artifact.

Open index.html directly after building, or host this folder as static files.

How to use

1. Open index.html in a modern browser.
2. Drag one or more BC Hydro consumption CSV files into the upload area.
3. If the same meter is split across different files, upload all files at once. The app will merge them, sort the intervals, resolve exact overlapping duplicate intervals where possible, and report gaps or conflicts.
4. Select the meter group and analysis mode.
5. Select a baseline option if you want differences shown against something other than RS 1101.
6. Review the annual comparison table, cost chart, monthly load profile, monthly cost breakdown, and gap report.
7. Export the summary CSV if desired.

Validation logic

- Requires a continuous, complete 365-calendar-day period before calculating the annual comparison.
- Reconstructs hourly records by meter and account number.
- Accounts for Pacific daylight saving time conventions used in BC Hydro data:
  - second Sunday in March has no 02:00 interval;
  - first Sunday in November has two 01:00 intervals.
- Reports missing intervals, unexpected intervals, non-hourly timestamps, incomplete dates, and unresolved overlapping files.
- Reports estimated-usage flags without rejecting those intervals automatically.
- Reports outflow or negative net consumption. Negative net intervals are capped at 0 kWh for these non-generation comparisons and are not treated as export credits.

Default rate assumptions

The default rates are editable in the website:

- RS 1101 basic charge: 23.44 cents/day
- RS 1101 Step 1: 11.87 cents/kWh
- RS 1101 Step 2: 14.08 cents/kWh
- RS 1101 Step 1 allowance: 675 kWh/month, converted to 22.1918 kWh/day
- RS 1151 basic charge: 25.00 cents/day
- RS 1151 energy: 12.70 cents/kWh
- RS 2101 overnight credit: minus 5.00 cents/kWh, 23:00 to 07:00
- RS 2101 on-peak charge: plus 5.00 cents/kWh, 16:00 to 21:00
- RS 2101 off-peak adjustment: 0.00 cents/kWh
- Deferral Account Rate Rider: minus 1.50% on base residential charges
- Trade Income Rate Rider: 0.00% on base residential charges
- Tax estimate: off by default, editable if required

Calculation notes

- Net Consumption (kWh) is used for cost calculations. If that column is blank, the app falls back to Inflow minus Outflow.
- The time-of-day adjustment is applied to all applicable hourly net consumption.
- The deferral and trade income riders are applied to RS 1101 and RS 1151 base charges, not to the RS 2101 time-of-day adjustment.
- The default comparison window is the latest continuous 365-day period.
- The annualized mode uses the longest complete continuous run and annualizes the cost to 365 days.
- Result exports include difference from the cheapest option, difference from the selected baseline option, and effective cents per kWh.

Validation

From this folder, run:

- npm run build
- npm run build:pages
- npm test
- npm run test:examples
- npm run test:browser

The synthetic, example, and browser smoke tests build TypeScript first. The synthetic test uses generated fixtures. The example and browser smoke tests read any local CSV files in the sibling examples folder without hard-coding their names or values.

Deployment

Pushes to the main branch run the GitHub Actions workflow at ../.github/workflows/pages.yml. The workflow installs dependencies, builds dist/, runs tests, checks the built page in Chromium, and deploys dist/ to GitHub Pages.

Privacy

All parsing and calculations run in the browser. The website does not transmit the CSV contents anywhere.
